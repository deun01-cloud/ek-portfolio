# 🌥 STEP 1. Cloudflare Worker 배포 (5분)

**왜 필요한가?** Yahoo Finance 시세 데이터를 가져오려면 작은 프록시 서버가 필요합니다. Cloudflare Workers는 이걸 무료로 제공해줘요. 카드 등록 불필요.

## 1-1. Cloudflare 계정 가입

1. https://dash.cloudflare.com/sign-up 접속
2. **이메일 + 비밀번호** 입력 → **Sign Up**
3. 이메일로 인증 메일 옴 → 클릭해서 인증
4. 로그인 완료

> 💡 가입 시 신용카드 정보 안 물어봅니다. 만약 물어보면 "무료 플랜"으로 가야 합니다.

## 1-2. Worker 생성

1. 로그인된 대시보드에서 **좌측 메뉴** → **Compute (Workers)** 또는 **Workers & Pages** 클릭
2. 우측 상단 **Create** 버튼 클릭
3. **Start with Hello World!** 카드의 **Get started** 클릭
4. **Worker 이름** 입력 (예: `yahoo-proxy`)
   - 이 이름이 URL의 일부가 됩니다 (예: `yahoo-proxy.당신ID.workers.dev`)
   - 이미 사용 중인 이름이면 다른 걸로 시도
5. 우측 하단 **Deploy** 버튼 클릭
6. "Successfully deployed" 메시지 확인

## 1-3. 코드 교체

1. 배포 완료 화면에서 **Edit code** 또는 **Continue to project → Edit code** 클릭
2. 좌측 코드 편집기에 기본 Hello World 코드가 있음
3. **편집기 안의 모든 코드를 전체 선택 후 삭제** (Ctrl+A → Delete)
4. 첨부드린 `worker.js` 파일을 텍스트 편집기로 열어서 **전체 내용 복사**
5. Cloudflare 편집기에 **붙여넣기**
6. 우측 상단 **Deploy** 버튼 클릭
7. "Deployed" 확인

## 1-4. 워커 URL 확인

배포 후 화면 상단에 URL이 표시됩니다:
```
https://yahoo-proxy.당신ID.workers.dev
```

**이 URL을 메모장에 복사해두세요!** Step 3에서 사용합니다.

## 1-5. 작동 테스트

브라우저 새 탭에서 다음 URL 접속:
```
https://yahoo-proxy.당신ID.workers.dev/yahoo/chart/005930.KS
```

✅ JSON 데이터가 나오면 성공. 이런 식으로 시작합니다:
```json
{"chart":{"result":[{"meta":{"currency":"KRW","symbol":"005930.KS",...
```

❌ 에러가 나면:
- "URL not found" → 워커 이름 + 경로 다시 확인
- "fetch failed" → 워커 코드가 제대로 붙여졌는지 확인

# 💻 STEP 2. 자산관리 툴 호스팅 (선택)

두 가지 옵션 중 하나 선택하세요.

## 옵션 A. 로컬 사용 (가장 간단, 1분)

가장 쉬운 방법. **그냥 `index.html` 파일을 더블클릭**하면 끝.

### 장점
- 설정 거의 없음
- 즉시 사용 가능

### 단점
- 그 PC에서만 사용 가능 (다른 PC/모바일에서 접속 불가)
- 데이터는 그 브라우저의 localStorage에만 저장
- PC 바꾸려면 데이터 백업/복원 필요 (설정 → JSON 내보내기/가져오기)

## 옵션 B. GitHub Pages (어디서든 접속, 5분)

### 2B-1. GitHub 가입 / 로그인
- https://github.com → 가입 또는 로그인 (무료)

### 2B-2. 새 리포지토리 생성
1. 우상단 **+** → **New repository**
2. Repository name: `ek-portfolio` (자유)
3. **Public** 선택 (GitHub Pages 무료는 Public 필수)
4. **Create repository**

### 2B-3. 파일 업로드
1. 새 리포지토리 화면에서 **uploading an existing file** 링크 클릭
2. `index.html` 파일을 드래그해서 업로드
3. 하단 **Commit changes**

### 2B-4. Pages 활성화
1. 리포지토리 상단 **Settings** 탭 클릭
2. 좌측 **Pages** 메뉴 클릭
3. Source: **Deploy from a branch**
4. Branch: **main** / **/(root)** → **Save**
5. 1~2분 대기 후 페이지 상단에 URL 표시:
   ```
   https://당신ID.github.io/ek-portfolio/
   ```

이 URL을 즐겨찾기에 추가하세요. 이게 본인 자산관리 툴 주소입니다.

# ⚙ STEP 3. 툴 초기 설정 (1분)

배포한 자산관리 툴 페이지로 접속하세요 (옵션 A면 더블클릭한 페이지, 옵션 B면 GitHub Pages URL).

## 3-1. 시뮬레이션 데이터 제거

처음 들어가면 시뮬레이션 데이터(가짜 계좌/종목)가 보입니다. 이걸 비우고 시작하세요:

1. 상단 탭 **⚙ 설정** 클릭
2. 페이지 하단 **데이터 관리** 카드 → **⚠ 전체 초기화** 클릭
3. 확인 두 번 (의도된 안전장치)
4. 페이지 새로고침 → 빈 상태로 시작

> 💡 새로고침 후에도 시뮬이 또 뜨면 정상입니다. 본인이 계좌를 1개라도 등록하면 더 이상 시뮬이 자동으로 뜨지 않아요.

## 3-2. 워커 URL 입력

1. **⚙ 설정** 탭
2. **API 연결** 카드 → **Cloudflare Worker 프록시 URL** 입력란에 Step 1에서 메모해둔 URL:
   ```
   https://yahoo-proxy.당신ID.workers.dev
   ```
   ⚠️ 끝에 `/` 절대 붙이지 마세요!

3. **세율 / 수수료** 카드:
   - 한국 양도세: 보통 **0** (대주주만 22%)
   - 미국 양도세: **22**
   - 배당소득세: **15.4**
   - 거래 수수료: 본인 증권사 수수료 (보통 **0.015**)

4. **벤치마크 & 환율** 카드:
   - 한국 벤치마크: `^KS11` (KOSPI)
   - 미국 벤치마크: `^GSPC` (S&P 500)
   - USD/KRW 환율: 현재 환율 입력 (예: `1380`)

5. 우측 하단 **💾 모든 설정 저장**

# 🚀 STEP 4. 실제 데이터 입력

## 4-1. 계좌 등록 (계좌 관리 탭)

**+ 계좌 추가** 버튼:

| 계좌 예시 | 종류 | 소유자명 | 증권사 |
|---|---|---|---|
| NH 본인계좌 | 본인 | 나 | NH투자증권 |
| 대신 본인계좌 | 본인 | 나 | 대신증권 |
| NH 자녀1계좌 | 자녀 | 김첫째 | NH투자증권 |
| NH 자녀2계좌 | 자녀 | 김둘째 | NH투자증권 |
| 회사 법인계좌 | 회사 | 이케이아이엔티 | (사용 증권사) |

> 💡 같은 사람이 여러 증권사 계좌를 가지고 있어도 **소유자명을 같게** 입력하면 대시보드에서 한 탭에 합산됩니다.

> 💡 예수금은 일단 0으로 두세요. CSV 임포트 시 자동으로 갱신됩니다.

## 4-2. 증권사 CSV로 거래내역 한 번에 임포트

### NH투자증권(나무증권) CSV 다운로드

1. https://www.nhsec.com 접속 (PC 환경)
2. **ID 로그인** (인증서 로그인 ❌ — ID 로그인이어야 함)
3. 메뉴: **뱅킹/계좌정보** → **거래내역** → **종합거래내역**
4. 계좌 / 조회 기간 (예: 1년) / 상품 구분 = "주식" 선택
5. 조회 후 **엑셀 저장** 버튼
6. `.xlsx` 파일이 다운로드됨

### 대신증권 (크레온) CSV 다운로드

1. 크레온 HTS 실행 (Windows PC 전용)
2. 로그인 후 화면 좌상단 **메뉴 검색**에 "거래내역" 입력
3. **거래내역조회** 또는 **종합거래내역** 화면 선택
4. 계좌 / 기간 선택 → 조회
5. 화면 우측 **엑셀** 또는 **저장** 버튼 → `.xlsx` 저장

> 💡 Mac 사용자는 크레온 HTS 사용 불가. 대신증권 홈페이지나 모바일 MTS의 거래내역 조회 후 캡처/수기 입력하거나, 한 번 Windows PC에서 받아두세요.

### 임포트 실행

1. 자산관리 툴 → **거래내역** 탭
2. 우상단 **📥 증권사 CSV 가져오기** 클릭
3. **대상 계좌**: 위에서 다운로드한 CSV의 출처 계좌 선택 (⚠️ 정확히!)
4. **파일 선택**: 다운로드한 .xlsx 파일
5. **📥 가져오기** 클릭
6. 토스트 알림으로 결과 확인:
   ```
   ✓ 가져오기 완료 — 매매 +47건 · 입출금 12건 · 예수금 갱신 (2,500,000원)
   ```

### 자동 처리되는 것
- ✅ 매수/매도 거래내역 추가
- ✅ 보유 종목 자동 갱신 (평균 매수가 자동 계산)
- ✅ 매도 시 실현손익 자동 계산 (세금/수수료 반영)
- ✅ 예수금 자동 갱신
- ✅ 중복 거래 자동 건너뜀 (다음에 또 임포트해도 안전)

## 4-3. 종목별 목표가/손절가 설정

CSV로 들어온 보유 종목들에 매도 타이밍을 설정해두세요:

1. **포트폴리오** 탭 (보유 종목 영역)
2. 종목 행의 **수정** 버튼
3. 입력:
   - 🎯 목표가 (예: +30%)
   - 🛑 손절가 (예: -10%)
   - 매수 논리 메모

도달 시 대시보드 알림 자동 표시됩니다.

## 4-4. 자산 배분 룰 (포트폴리오 탭 상단)

본인의 투자 원칙을 룰로 등록하면, 비중 초과 시 알림 받습니다:

- 산업 Technology 50% 이내 (반도체 쏠림 방지)
- 단일 종목 NVDA 15% 이내 (개별 종목 리스크 관리)
- 본인계좌 35% (자산 배분)

## 4-5. 관심종목 / 매수후보 (퀀트 분석 탭)

매수 검토 종목들을 미리 등록해두면 가격 도달 시 알림:

- **+ 종목 추가** 버튼
- 상태 선택:
  - 🟢 **검토 중**: 분석 끝, 매수 직전
  - 🟡 **모니터링**: 목표 매수가 입력 → 도달 시 자동 알림
  - 🔵 **관심**: 일단 찜만

# 🆘 문제 해결

## "Cloudflare 가입 시 카드 정보를 요구함"
- 무료 플랜은 카드 불필요. 가입 페이지를 잘 보면 "Free" 옵션이 따로 있습니다.

## "워커 URL을 입력했는데 시세가 안 나옴"
- 설정 화면의 URL 끝에 `/` 가 없는지 확인
- 워커 URL을 브라우저에서 직접 열어서 `/yahoo/chart/005930.KS` 붙여 보고 JSON이 나오는지 확인
- 헤더의 `Yahoo Finance` 표시등이 빨간색이면 워커 연결 실패

## "GitHub Pages 페이지가 404"
- Save 후 1-2분 기다려야 함
- Source가 `main` 브랜치 `/(root)`로 설정됐는지 확인
- 리포지토리가 Public인지 확인

## "CSV 임포트 결과가 이상함"
- 한국 증권사들 CSV 형식이 회사마다 다양해서 100% 자동 인식이 어려울 수 있음
- 임포트 결과가 0건이거나 컬럼이 잘못 매칭되면, **실제 받으신 CSV 파일을 보여주시면** 그 형식에 맞춰 파서 조정 가능
- KOSDAQ 종목이 .KS로 잘못 등록되면 보유종목 편집에서 .KQ로 수동 변경 가능

## "데이터를 다른 PC로 옮기고 싶음"
- 기존 PC: 설정 → 📥 데이터 내보내기 (JSON) → 파일 다운로드
- 새 PC: 설정 → 📤 데이터 가져오기 → 그 JSON 파일 선택

## "시뮬레이션 데이터가 계속 뜸"
- 본인이 계좌를 1개라도 추가하면 자동 시드가 더 이상 작동하지 않음
- 만약 계속 뜨면 브라우저 → F12 (개발자 도구) → Application → Local Storage → ekAssetTool 키 삭제
